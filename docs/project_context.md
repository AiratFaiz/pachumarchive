# Pachuca Archive — актуальный контекст проекта

## Цель проекта

Создать удобный архив контента Пачуки с поиском, фильтрацией и ссылками на оригинальные источники.

Основной сценарий:

* найти конкретный контент;
* посмотреть, смотрел ли его Пачука;
* перейти на запись на YouTube, Telegram или Boosty;
* в будущем объединять одинаковый контент с разных площадок в одну карточку.

# Скрипты

## collect_new_content.py

Собирает новые записи из:

* YouTube
* Boosty

Добавляет только отсутствующие записи в:

```text
content_drafts.csv
```

Проверяет наличие по URL.

---

## collect_telegram_drafts.py

Берёт:

```text
result.json
```

из Telegram Desktop.

Добавляет последние записи в:

```text
content_drafts.csv
```

Не дублирует уже существующие записи.

---

## apply_drafts.py

Берёт строки со статусом:

```text
approved
```

из:

```text
content_drafts.csv
```

и переносит их в:

```text
content_items.csv
```

После импорта меняет статус на:

```text
imported
```

---

# Модерация

Все новые записи сначала попадают в:

```text
content_drafts.csv
```

Перед импортом вручную проверяются:

```text
suggested_canonical_title
suggested_content_type
suggested_content_tags
```

После проверки:

```text
status = approved
```

и запись импортируется.

## Строение content_cards.csv

Минимальная актуальная структура:

* card_id — уникальный ID карточки
* card_title — красивое название карточки
* content_type — game / movie / series / anime / battle_rap / music / etc
* content_tags — теги карточки
* rating — итоговая оценка, если есть
* rating_source — boosty / telegram / manual
* is_active — 1/0, чтобы скрывать карточку без удаления

Остальные поля можно добавить позже при необходимости.


## Строение content_card_items.csv

Минимальная актуальная структура:

* card_id — ID карточки
* content_id — ID строки из content_items.csv
* sort_order — порядок материала внутри карточки
* card_name — служебное поле для удобства ручной работы

`card_name` не является источником истины и может игнорироваться сайтом.
Основное название карточки хранится в `content_cards.card_title`.

# Модель карточек контента

Важно: `content_items.csv` остается первичной сущностью проекта.

Одна строка в `content_items.csv` = один конкретный материал с площадки:

* видео YouTube;
* пост Boosty;
* перезалив Telegram;
* отдельная запись стрима;
* нарезка;
* реакция;
* прохождение и т.д.

`content_cards.csv` — это надстройка над частью контента, а не замена основной таблицы.

Карточки нужны, чтобы объединять несколько материалов вокруг одной сущности, которую пользователь реально будет искать.

Примеры карточек:

* Resident Evil 7
* Во все тяжкие
* Сверхъестественное
* 007 First Light
* Кубок МЦ
* Натальная карта с Харламовым

Не обязательно, чтобы 100% записей имели карточку.

Часть контента может остаться только в `content_items.csv` и находиться через обычный поиск.

Это нормально.

Например:

* случайный ролик;
* разовый заказ;
* небольшой фрагмент стрима;
* видосы без понятной самостоятельной сущности;
* контент, который пока не хочется группировать.

Если позже появится необходимость, для такой записи всегда можно создать карточку и связать её через `content_card_items.csv`.

---

## Принцип карточек

Карточка = сущность контента.

Материал = конкретная публикация на площадке.

Например:

```text
Карточка:
007 First Light

Материалы:
- 007 First Light #1-2
- 007 First Light #3-4 + Видосы Drago
