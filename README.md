# Pachuca Archive

Архив контента Пачуки: сайт для поиска просмотренного и пройденного контента с переходом к оригинальным материалам на YouTube, Telegram и Boosty.

Сейчас проект решает три задачи:

* собирает материалы с разных площадок;
* приводит их к единой CSV-модели;
* показывает пользователю карточки контента с поиском, фильтрами и ссылками на источники.

## Сайт

Текущая главная страница строится вокруг карточек контента.

Карточка — это сущность, которую пользователь реально ищет: игра, фильм, сериал, аниме, баттл, подборка или другой самостоятельный контент. Внутри карточки могут быть материалы с разных площадок: полный стрим, перезалив, нарезка, реакция или отдельная запись.

Примеры карточек:

* `Resident Evil 7`
* `Во все тяжкие`
* `Сверхъестественное`
* `007 First Light`

## Поиск

Поиск работает по карточной модели, а не по сырым строкам CSV.

Результаты делятся на два уровня:

* `strong` — запрос совпал с самой карточкой: названием, типом, тегом или человекочитаемым названием тега;
* `weak` — запрос совпал только с вложенными материалами карточки.

Если карточка найдена как `weak`, внутри неё показываются только совпавшие материалы, а не весь список. Дубли между `strong` и `weak` убираются по `content_id`, чтобы один и тот же материал не показывался дважды в разных карточках.

Подробные правила поиска: [`docs/content_card_design.md`](docs/content_card_design.md#поиск).

## Структура проекта

```text
src/app/                 страницы Next.js
src/components/          UI-компоненты
src/lib/                 чтение и подготовка данных для сайта
public/data/             CSV, которые использует сайт
scripts/                 рабочие скрипты сбора и импорта данных
docs/                    документация по проекту и модели данных
data/                    черновики и рабочие файлы модерации
```

## Основные данные

Сайт использует три основные таблицы:

```text
public/data/content_items.csv
public/data/content_cards.csv
public/data/content_card_items.csv
```

`content_items.csv` хранит конкретные материалы с площадок.

`content_cards.csv` хранит карточки контента.

`content_card_items.csv` связывает карточки с материалами.

Подробнее: [`docs/content_card_design.md`](docs/content_card_design.md).

## Локальный запуск

Установить зависимости:

```bash
npm install
```

Запустить dev-сервер:

```bash
npm run dev
```

Проверить код линтером:

```bash
npm run lint
```

Проверить структуру CSV-данных:

```bash
npm run validate:data
```

Собрать production-версию:

```bash
npm run build
```

## Обновление данных

Новые материалы сначала попадают в черновики, затем вручную проверяются и импортируются в основную таблицу.

Общая схема:

```text
YouTube / Boosty / Telegram
        ↓
data/content_drafts.csv
        ↓
ручная модерация
        ↓
public/data/content_items.csv
        ↓
карточки и связи
        ↓
сайт
```

Основные команды:

```bash
python scripts/collect_new_content.py
python scripts/apply_drafts.py
```

Подробный контекст проекта: [`docs/project_context.md`](docs/project_context.md).

## Текущий статус

Проект находится в стадии наполнения и очистки данных. Интерфейс уже использует карточную модель, но часть данных ещё может содержать дубли, неполные оценки, неточные теги или не до конца объединённые материалы.
