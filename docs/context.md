# Контекст: баттлрэп, теги и фильтрация

## Решение

Для баттлрэпа не стоит пытаться покрыть всё карточками.

Причина: один стрим часто содержит несколько баттлов, площадок, сезонов, ивентов и форматов. Если делать карточки на каждую сущность, быстро появится много шумных many-to-many связей и карточки станут менее полезными.

Лучший подход для баттлрэпа:

* карточки оставлять только для крупных пользовательских сущностей;
* основную навигацию строить через теги и фасетные фильтры.

## Когда нужны карточки

Карточки стоит делать для крупных ретроспектив, сезонов, подборок или понятных самостоятельных сущностей.

Примеры:

* `SLOVOSPB 2 | Ретроспектива`
* `SLOVOSPB 3 | Ретроспектива`
* `Fresh Blood 3 | Ретроспектива`
* `Fresh Blood 4 | Ретроспектива`
* `RBL Season 2 | Ретроспектива`
* `ProBattle | Судейство`

Карточку не стоит делать на каждый отдельный баттл, если он является только частью мультиконтентного стрима.

## Базовая модель тегов

Все остаётся в текущем поле `content_tags`.

Для баттлрэпа теги лучше использовать как несколько логических уровней:

* крупный тип: `battle_rap`
* площадка/проект: `kbk`, `rbl`, `slovo`, `140`, `versus`, `rnb`, `probattle`
* ивент/сезон: `kbk_reborn`, `kbk_new_era`, `kbk_summer_trip`, `rbl_universe`, `rbl_s2`, `slovospb_s2`
* формат: `retro`, `cypher`, `judging`, `reaction`, `tier_list`, `review`, `compilation`

Пример:

```text
battle_rap,kbk,kbk_reborn,cypher
battle_rap,kbk,kbk_new_era,judging
battle_rap,rbl,rbl_universe,reaction
battle_rap,rbl,rbl_s2,retro
battle_rap,slovo,slovospb_s2,retro
battle_rap,slovo,slovo_moscow_4,retro
battle_rap,140,retro
battle_rap,versus,fb,retro
```

## Naming Convention

Для площадок использовать короткие ключи:

```text
kbk
rbl
slovo
140
versus
rnb
probattle
```

Для ивентов и сезонов лучше использовать префикс площадки:

```text
kbk_reborn
kbk_new_era
kbk_summer_trip
rbl_universe
rbl_s2
slovospb_s2
slovospb_s3
slovo_moscow_4
```

Почему лучше `kbk_reborn`, а не просто `reborn`:

* меньше риск конфликтов;
* легче понять тег без контекста;
* проще строить зависимые фильтры;
* проще валидировать;
* проще найти все ивенты Кубка МЦ по `kbk_`.

Форматы держать отдельными тегами:

```text
retro
cypher
judging
reaction
tier_list
review
compilation
```

Не стоит склеивать все уровни в один тег:

```text
kbk_reborn_cypher
```

Лучше так:

```text
kbk,kbk_reborn,cypher
```

Так потом можно независимо фильтровать:

* весь Кубок МЦ;
* только Reborn;
* только cypher;
* Reborn + cypher.

## UI-фильтрация

Для вкладки `Баттлрэп` используются отдельные фасетные фильтры.

Пример групп:

```ts
const battleRapFilters = {
  platforms: ["kbk", "rbl", "slovo", "140", "versus", "rnb", "probattle"],
  events: ["kbk_reborn", "kbk_new_era", "kbk_summer_trip", "rbl_universe", "rbl_s2", "slovospb_s2"],
  formats: ["retro", "cypher", "judging", "reaction", "tier_list", "review", "compilation"],
};
```

В UI это может выглядеть так:

```text
Площадка:
Все / Кубок МЦ / RBL / SLOVO / 140 BPM / Versus / Рвать на битах

Ивент:
Все / Reborn / New Era / Summer Trip / RBL Universe / RBL Season 2 / SLOVOSPB 2

Формат:
Все / Ретроспектива / Cypher / Судейство / Реакция / Тир-лист
```

Ивенты можно делать зависимыми от выбранной площадки. Например, если выбрана площадка `Кубок МЦ`, показывать только `kbk_*` события.

Сейчас вложенные фильтры ивентов включены для `Кубок МЦ` (`kbk_*`) и `RBL` (`rbl_*`). Остальные баттл-площадки фильтруются только на уровне площадки, пока их ивенты не размечены достаточно последовательно.

Если фильтр по баттл-площадке или ивенту оставляет внутри карточки-контейнера один материал, UI показывает название, дату и теги этого материала. Это помогает не показывать общий заголовок контейнера вроде `Баттл-рэп: Главное`, когда пользователь фактически нашёл конкретный материал по `Кубок МЦ`.

При раскрытии карточки или группы источника внутри карточки UI сохраняет позицию раскрываемого заголовка на экране. Это нужно для больших карточек с длинными списками, чтобы пользователь не терял место клика после закрытия предыдущего раскрытого списка, но без принудительного скролла к началу блока.

## Важный принцип

Технически всё это обычные `content_tags`, но логически они относятся к разным группам.

Не стоит выводить все теги одним общим фильтром, иначе получится каша из площадок, сезонов, форматов и персон.

Лучше хранить группы тегов в коде отдельным словарём и использовать их для UI-фильтров.

## Рекомендация

Размечать баттлрэп по паттерну:

```text
battle_rap,<platform>,<platform_event>,<format>
```

Примеры:

```text
battle_rap,kbk,kbk_reborn,cypher
battle_rap,kbk,kbk_new_era,judging
battle_rap,rbl,rbl_universe,reaction
battle_rap,slovo,slovospb_s2,retro
```

Это хорошо масштабируется, не требует новой CSV-структуры и позволит позже сделать удобные фильтры без переделки данных.
