# Модель карточек контента

Этот документ описывает карточную модель Pachuca Archive: какие CSV участвуют, как они связаны и как эти данные отображаются на сайте.

Общий контекст проекта находится в [`project_context.md`](project_context.md).

## Основная идея

Пользователь ищет не конкретную строку из CSV, а понятную сущность: игру, фильм, сериал, аниме, баттл или подборку.

Поэтому сайт использует два уровня:

* материал — конкретная публикация на площадке;
* карточка — пользовательская сущность, объединяющая один или несколько материалов.

Пример:

```text
Карточка:
Resident Evil 7

Материалы:
полный стрим на Boosty
перезалив в Telegram
нарезка на YouTube
```

## CSV-файлы

### `content_items.csv`

Путь:

```text
public/data/content_items.csv
```

Одна строка = один материал.

Материалом может быть:

* видео YouTube;
* пост Boosty;
* перезалив Telegram;
* запись стрима;
* нарезка;
* реакция;
* прохождение;
* разовый заказ или другой самостоятельный фрагмент.

Ключевые поля:

* `content_id` — уникальный ID материала;
* `canonical_title` — нормальное название материала;
* `content_type` — тип контента;
* `content_tags` — теги через запятую;
* `rating` — оценка, если есть;
* `rating_source` — источник оценки;
* `first_date` — первая дата материала;
* `last_date` — последняя дата материала;
* `duration_hours` — длительность в часах;
* `youtube_url` — ссылка на YouTube;
* `telegram_url` — ссылка на Telegram;
* `boosty_url` — ссылка на Boosty.

### `content_cards.csv`

Путь:

```text
public/data/content_cards.csv
```

Одна строка = одна карточка контента.

Ключевые поля:

* `card_id` — уникальный ID карточки;
* `card_title` — название карточки для интерфейса;
* `content_type` — основной тип карточки;
* `content_tags` — теги карточки;
* `rating` — итоговая оценка карточки;
* `rating_source` — источник итоговой оценки;
* `is_active` — `1` или `0`, позволяет скрыть карточку без удаления.

### `content_card_items.csv`

Путь:

```text
public/data/content_card_items.csv
```

Таблица связей между карточками и материалами.

Ключевые поля:

* `card_id` — ID карточки из `content_cards.csv`;
* `content_id` — ID материала из `content_items.csv`;
* `sort_order` — порядок материала внутри карточки;
* `card_name` — служебное поле для ручной работы, не является источником истины.

## Важные правила модели

`content_items.csv` остаётся первичным источником материалов. Карточки не заменяют материалы, а группируют их для интерфейса.

Карточка может содержать один материал. В этом случае карточка на сайте сразу открывает источник.

Карточка может содержать несколько материалов. В этом случае карточка раскрывается и показывает список источников.

Один материал может входить в несколько карточек. Это нужно для мультиконтентных стримов, где в одной записи были, например, фильм и игра.

Не каждый материал обязан иметь ручную карточку. Если материал не связан с `content_cards.csv`, код сайта создаёт для него одиночную карточку автоматически.

Карточки с `is_active = 0` не показываются.

## Как сайт собирает карточки

Логика находится в:

```text
src/lib/contentCards.ts
```

Сборка происходит так:

1. Читаются `content_items.csv`, `content_cards.csv` и `content_card_items.csv`.
2. Материалы индексируются по `content_id`.
3. Активные карточки связываются с материалами через `content_card_items.csv`.
4. Материалы внутри карточки сортируются по `sort_order`, затем по дате.
5. Материалы без ручной карточки превращаются в одиночные карточки.
6. Итоговый список сортируется по самой поздней дате материала внутри карточки.

## Отображение на сайте

Основной компонент:

```text
src/components/cardsView.tsx
```

Сетка:

* desktop — две карточки в ряд;
* mobile — одна карточка в ряд.

На карточке показываются:

* тип контента;
* дата последнего обновления;
* название;
* оценка, если есть;
* теги;
* иконки доступных площадок.

## Сценарии карточек

### Один материал

Если после фильтрации в карточке остался один материал, клик по карточке открывает источник в новой вкладке.

### Несколько материалов

Если материалов несколько, карточка раскрывается и показывает список материалов.

### Несколько источников

Если внутри карточки есть несколько площадок и на одной или нескольких площадках больше одного материала, материалы группируются по источникам.

## Фильтрация и поиск

Поиск проверяет:

* название карточки;
* тип контента;
* оценку;
* теги;
* названия материалов внутри карточки.

Фильтр по площадке работает по материалам внутри карточки. Если для выбранной площадки у карточки нет материалов, карточка скрывается.

Фильтр по типу работает по `content_type` карточки.

## Сортировка

Поддерживаются варианты:

* новые сверху;
* старые сверху;
* название от А до Я;
* название от Я до А.

Для сортировки по дате используется максимальная дата среди материалов карточки:

```text
MAX(last_date или first_date)
```

## Рейтинг

Рейтинг хранится на уровне карточки в `content_cards.csv`.

Если материал не имеет ручной карточки и превращается в одиночную карточку автоматически, рейтинг берётся из самого материала.

## Что пока не является частью модели

Сейчас не используются:

* `relation_type`;
* `display_title` для связи;
* `is_primary`;
* отдельные страницы карточек как основной сценарий;
* обложки карточек;
* рекомендации похожего контента;
* админка для модерации карточек.

Эти поля и сценарии можно добавить позже, когда появится реальная необходимость.

## Риски и проверки данных

Для стабильной работы сайта желательно автоматически проверять:

* ссылки из `content_card_items.csv` на отсутствующие `content_id`;
* ссылки на отсутствующие `card_id`;
* дубли тегов внутри одной карточки;
* неизвестные значения `content_type`;
* пустые `card_title` и `canonical_title`;
* невалидные даты;
* материалы без ссылок на площадки.
